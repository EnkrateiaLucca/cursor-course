The #1 Mistake

Jumping straight to code.

The AI has no idea what your codebase looks like,
what patterns you use, or what you actually need.

More context ≠ better results — curate what the AI sees.

The Framework

🔍

Explore

Understand the codebase. Discover patterns, structure, and conventions before touching anything.

📋

Plan

Define what to build and how. Get a roadmap from the AI — files, data flow, architecture.

🛠️

Build

Execute the plan with Agent mode. The AI writes code that matches your existing patterns.

Your Context Navigator: @

The @ symbol is how you point Cursor at exactly the right context.

Symbol What It Does When to Use
@file Reference a specific file Precise edits, explaining code
@folder Reference a directory Understanding module structure
@codebase Semantic search across project Discovery, "where is X handled?"
@docs Query documentation Framework APIs, library usage
@web Search the web Latest best practices, new APIs

@ Symbol in Action

Explore architecture

@codebase What is the overall architecture of this project?
What framework does it use, and how are files organized?

Inspect a file

@app/page.tsx Explain what this page does and how it's structured.

Scan a folder

@components/ What components exist? Give me a summary of each.

Search the web

@web What are the latest best practices for Next.js Server Actions?

The Context File

A single markdown file that serves as your project's "north star".

Without It

  • Repeat project details in every prompt
  • AI guesses your tech stack
  • Inconsistent output across sessions

With It

  • One @context.md gives full picture
  • AI knows stack, patterns, constraints
  • Consistent, aligned code every time

Context File Template


# [Project Name] — Context File

## Problem Statement
[What problem does this app solve? Who is it for?]

## Core Features
1. **[Feature]** — Brief description
2. **[Feature]** — Brief description

## Tech Stack
| Layer      | Technology         | Why                  |
|------------|--------------------|----------------------|
| Framework  | Next.js 15         | Server components    |
| Styling    | Tailwind + shadcn  | Rapid UI development |
| Database   | Supabase           | Managed DB + auth    |
| Auth       | Clerk              | Drop-in auth         |

## Key Routes
| Route        | Purpose             |
|--------------|---------------------|
| /            | Landing page        |
| /dashboard   | Main user dashboard |

## Design Principles
- Server components by default
- Mobile-first responsive design
- All API calls through Server Actions

Save as context.md → reference with @context.md in every prompt

Explore Phase

Understand the template before writing a single line.

1 Open McKay's template in Cursor
2 Ask: @codebase What is the file structure? What patterns does it use?
3 Explore app/, components/, lib/
4 Check routing patterns and existing pages

Plan Phase

Get a roadmap before touching code.


@codebase I want to add a simple quiz page to this app.

The quiz should:
- Show one question at a time with 4 multiple-choice answers
- Track score as the user progresses
- Show results at the end
- Use hardcoded questions (no AI, no API)

Create a step-by-step plan. Don't write code yet —
just outline the files to create/modify.

Review the plan. Adjust if needed. Then build.

Build Phase

Execute the plan with Agent mode.

1 Switch to Agent mode and implement the plan
2 Run pnpm dev and verify the quiz
3 Iterate — use Cmd+K inline edits for quick fixes
4 Add navigation links to the quiz page

Build is fast because the AI already has good context from Explore + Plan.

Key Takeaways

  • Explore prevents building something that doesn't match existing patterns
  • Plan gives you a clear roadmap before writing any code
  • Build is fast because the AI has good context
  • @file for precision, @codebase for discovery
  • A context file saves you from repeating yourself every prompt
