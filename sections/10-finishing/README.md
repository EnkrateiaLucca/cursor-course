## Key Takeaways

1. **Cursor is a development multiplier, not a replacement for thinking** — you still make the architectural decisions
2. **Explore → Plan → Build** is the framework — don't skip the first two steps
3. **Context quality > prompt cleverness** — curate what the AI sees
4. **Project rules encode your team's standards** — add them incrementally, not all at once
5. **MCP extends Cursor's reach** — Playwright, GitHub, docs, databases
6. **The two-stage workflow works** — plan with ChatGPT/Claude, build with Cursor
7. **Test-driven development with AI** — write the test (define behavior), let AI implement
8. **Full-stack is achievable** — Supabase + Clerk + Stripe + Vercel = production app
9. **Git discipline is non-negotiable** — commit early, commit often
10. **Try 5 times before giving up** — AI prompting is a skill that improves with practice
