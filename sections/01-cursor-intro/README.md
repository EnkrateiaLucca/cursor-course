# Section 01: Cursor Intro

**Slides:** 1–6 | **Duration:** ~45 min | **Break after:** Yes

## Learning Goals

- Understand what Cursor is and how it differs from traditional IDEs
- Use all four core interaction modes: Chat, Agent, Inline Edit, Tab autocomplete
- Build a simple HTML app from scratch using Cursor
- Develop intuition for when to use each mode

## Prerequisites

- Cursor IDE installed and signed in
- No project setup needed (we start from an empty file)

## Key Talking Points

- Cursor is AI-first, not AI-bolted-on — built from the ground up for AI-assisted development
- The four modes serve different needs: quick edits vs. multi-file changes vs. exploration
- Emphasize the "try 5 times" rule — if a prompt doesn't work, rephrase and retry
- Git commits are critical — Cursor's checkpoint system is NOT a replacement for git
